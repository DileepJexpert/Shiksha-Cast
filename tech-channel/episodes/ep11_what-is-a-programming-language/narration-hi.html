EP11 — What is a programming language — and why so many?

HINGLISH (Roman)
01Title

Programming language hoti kya hai — aur itni saari kyun hain? Welcome to Katixo KhojLab. Computers sirf 0s aur 1s samajhte hain, phir bhi humne 700 se zyada languages banayi hain unse baat karne ke liye. Aaj pata karte hain — aisa kyun.

[CUE] Title pe ~2s ruko, calm curious tone, phir aage badho.
02The Hook

Pichle episode mein humne seekha ki CPU sirf 0s aur 1s samajhta hai — pure binary. Ab socho ek poori app raw 0s aur 1s mein likhni pade. Yeh toh bilkul torture ho jayega. Isliye humne 700 se zyada programming languages banayi. Par ek ajeeb sawaal uthta hai — itni saari kyun?

[CUE] Yahan energy high — ye hook hai. "torture" pe thoda wince karo.
03Aaj ka sawaal

Toh aaj ka ek sawaal: agar computer sirf binary bolta hai, toh humein code likhne ke itne saare alag-alag tareeke kyun chahiye? Chalo isse todte hain.

[CUE] Thoda slow. Sawaal ko land hone do, phir aage.
04Idea 1 — Human-friendly bridge

Idea number one. Programming language bas ek human-readable tareeka hai computer ke liye instructions likhne ka. Yeh line dekho — print, bracket kholo, quote, Namaste, quote, bracket band. Ise aap sach mein padh sakte ho. Yeh ek bridge hai: aap kuch clear likhte ho, aur baad mein wo translate hoke 0s aur 1s ban jaata hai jo CPU chalata hai.

[CUE] Code line pe point karo. Stress ki ek insaan ise padh sakta hai.
05Idea 2 — Compiler / interpreter

Par yeh translate karta kaun hai? Ek special program — jise compiler kehte hain, ya interpreter. Aap likhte ho print Namaste, aur translator use exact binary instructions mein badal deta hai jo processor chalata hai. Sabse achhi baat — aap kabhi khud 0s aur 1s ko haath nahi lagate. Translator wo sab sambhal leta hai.

[CUE] Arrow reveal karo: code → translator → binary, left to right.
06Idea 3 — Itni saari kyun: alag jobs

Ab badi baat — itni saari languages kyun? Kyunki alag-alag kaam ke liye alag tools chahiye. Python simple hai — AI aur quick scripts ke liye best. JavaScript aapke browser mein websites chalata hai. C aur C plus plus bahut fast hain — games aur systems ke liye. Aur SQL databases se baat karne ke liye. Yeh ek trade-off triangle hai: easy versus fast versus safe — koi ek language teeno mein nahi jeetti.

[CUE] Python, JavaScript, C/C++, SQL bolte waqt har chip reveal karo.
07Idea 4 — Same core ideas

Ab wo part jisse sab click ho jaata hai. Andar se, har language same basic ideas express karti hai — variables, if-else decisions, loops, aur functions. Bilkul waise jaise Hindi, English aur Tamil sab same baat alag shabdon mein kehte hain. Toh ek baar aap ek language sach mein seekh lo, toh agli zyada-tar bas naya vocabulary hoti hai.

[CUE] Language wali analogy dheere se land karo — ye reassuring beat hai.
08Why it matters

Toh ye aapke liye kyun matter karta hai? Ye aapko apne goal ke liye sahi language chunne mein madad karta hai. Ye samjhata hai ki ek hi app aksar kai languages saath kyun use karti hai — ek website ke liye, ek server ke liye, ek database ke liye. Aur ye us classic panic ko shaant karta hai — "pehle kaunsi language seekhun?" Sachchi baat: almost koi bhi, kyunki concepts transfer ho jaate hain.

[CUE] "pehle kaunsi" wali line pe warm, reassuring tone.
09Common myth

Aur isse aata hai sabse bada myth. Log sochte hain ek hi best language hai, aur aapko sab master karni padti hain. Aisa nahi hai. Har language ek tool hai jo apne job ke liye banaya gaya hai, aur sab same core logic share karti hain. Toh hype ke peeche mat bhaago — jo bana rahe ho uske hisaab se chuno.

[CUE] Myth pe ek beat ke baad green "Reality" card reveal karo.
10Recap

Quick recap. Ek: programming language bas human-friendly instructions hai, jo translate hoke machine code ban jaati hai. Do: itni saari isliye hain kyunki alag kaam ki alag zaroorat hai — par core ideas sab mein same hain.

[CUE] Crisp aur clear — ye do lines log yaad rakhein.
